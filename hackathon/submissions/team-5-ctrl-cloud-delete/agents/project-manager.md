# Project Manager

## Hackathon: Contoso Financial — Cloud Migration

---

## Responsibilities

- Own the overall delivery timeline and keep the team on track within the hackathon window
- Manage the CFO/CTO stakeholder tension — ensure every architectural decision is documented with a clear rationale
- Track which migration pattern (rehost / re-platform / refactor) has been chosen for each workload and why
- Coordinate handoffs between Architect, Developer, and Data Engineer
- Own the final migration rationale document and presentation narrative
- Flag blockers early and reprioritise scope if time is short

---

## Key Outputs

- Migration rationale document (which cloud, which pattern per workload, trade-off justification)
- Demo script and talking points for the final presentation

## Stakeholder Position

| Stakeholder | Priority | Preference |
|-------------|----------|------------|
| CFO | Speed — get off prem fast | Rehost / lift-and-shift |
| CTO | Modernisation — cloud-native | Re-platform / refactor |

The PM must ensure the final solution explicitly addresses both positions and justifies any trade-offs made.
